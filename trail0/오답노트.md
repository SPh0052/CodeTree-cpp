## Chapter 1 — 변수 · char/string · cout (2026-07-14)

### ① 문자(char) vs 문자열(string)

```cpp
char a = 'T';      // 문자 하나 → 작은따옴표 ' '
string a = "T";    // 문자열   → 큰따옴표 " "
```

- **문자(char)** 는 딱 한 글자, **작은따옴표** `'T'`
- **문자열(string)** 은 0글자 이상, **큰따옴표** `"T"`

---

### ② 같은 이름으로 변수를 두 번 선언하면 안 되는 이유

#### ❌ 틀린 코드

```cpp
int main() {
    char a = 'C';
    char a = 'T';   // ❌ 컴파일 에러: redeclaration of 'char a'
    cout << a;
    return 0;
}
```

#### 🐛 왜 안 될까?

`char a = 'C';` 이 한 줄은 사실 두 가지 일을 한다.

1. **선언(declaration)** — "메모리에 `a`라는 이름의 char 공간을 하나 잡아라"
2. **초기화(initialization)** — "거기에 `'C'`를 넣어라"

그런데 다음 줄에서 또 `char a`라고 하면, 컴파일러 입장에선 **이미 있는 이름으로 또 공간을 잡으라**는 소리가 된다. 그럼 나중에 `a`라고 썼을 때 둘 중 뭘 가리켜야 할지 알 수 없다. 그래서 아예 금지.

#### ✅ 값을 바꾸고 싶으면 (선언은 한 번만, 대입은 여러 번)

```cpp
int main() {
    char a = 'C';   // 선언 + 초기화 (한 번만)
    a = 'T';        // 재대입 (type 없이!)
    cout << a;      // T
    return 0;
}
```

> 💡 **핵심:** `char`(타입)는 **선언할 때 딱 한 번**만 쓴다. 값만 바꿀 땐 타입 없이 `a = 'T';`

---

### ③ 변수 값 대입 순서 이해하기

```cpp
#include <iostream>
using namespace std;

int main() {
    int a = 1;
    int b = 5;
    int c = 3;

    a = c;      // a = 3        (a에 c 값 복사)
    a = a + c;  // a = 3 + 3 = 6
    b = b - c;  // b = 5 - 3 = 2

    cout << a << "\n";  // 6
    cout << b << "\n";  // 2
    cout << c << "\n";  // 3  (c는 안 바뀜)
    return 0;
}
```

> 💡 대입(`=`)은 **오른쪽을 먼저 계산해서 왼쪽에 넣는다.** 윗줄부터 순서대로 값이 갱신된다.

---

### ④ C++ `cout` vs C `printf` — 설계 철학의 차이

> **의문:** "C++은 변수랑 줄바꿈, 문자를 한 따옴표 안에 동시에 표현할 수 없구나. C언어(`printf`)는 됐는데?"

정확히 이해한 것. `cout`은 그런 방식이 **아니다.**

| | C `printf` | C++ `cout` |
|---|---|---|
| 방식 | **틀(서식 문자열)** 을 주고 **구멍(`%d`)** 에 값을 꽂음 | 출력할 것들을 `<<`로 **순서대로 나열** |
| 예시 | `printf("%d\n", x);` | `cout << x << "\n";` |
| 타입 검사 | ❌ 못 잡음 (`%d`에 실수 넣어도 통과) | ✅ 컴파일러가 타입 보고 알아서 처리 |
| 서식 문자 | `%d %f %lf %c %s`… 외워야 함 | 필요 없음 |

**`cout`이 이렇게 설계된 이유:**
`printf`는 `%d`인데 실수를 넣거나 `%s`인데 정수를 넣어도 **컴파일러가 못 잡아낸다.** 그래서 실행 중에 이상한 값이 나오거나 프로그램이 터진다. 반면 `cout << x`는 컴파일러가 `x`의 타입을 보고 알아서 처리하니 그런 사고가 안 나고, 서식 문자(`%d`, `%f`, `%lf`, `%c`, `%s`…)를 외울 필요도 없다.

## Chapter 2 — 실수 출력 · precision / fixed (2026-07-14)

### ① `precision(n)`의 뜻 — `fixed` 유무에 따라 달라진다

`precision(n)`은 `fixed`가 있냐 없냐에 따라 **의미가 완전히 달라진다.**

| | `precision(n)`의 뜻 |
|---|---|
| **`fixed` 없음** (기본) | 전체 **유효숫자** n개 |
| **`fixed` 있음** | **소수점 아래** n자리 |

---

### ② 직접 비교: `a = 33.567268`

같은 값 `a = 33.567268`을 놓고 비교해보면 차이가 확 드러난다.

```cpp
// fixed 있음 → 소수점 아래 자릿수
precision(4) → 33.5673   // 소수점 아래 4자리
precision(2) → 33.57     // 소수점 아래 2자리

// fixed 없음 → 전체 유효숫자
precision(4) → 33.57     // 유효숫자 4개 (3,3,5,7)
precision(2) → 34        // 유효숫자 2개 (3,4)  ← !!
```

> ⚠️ `fixed` 없이 `precision(2)`를 주면 **`34`** 가 나온다. 소수점이 통째로 날아간다.
> **"정수 부분도 자릿수에 포함"** 되기 때문이다.

---

### ③ 왜 `fixed`가 아무 역할 안 하는 것처럼 보였나

우연히 결과가 겹쳐서 그렇게 보였을 뿐이다.

- `fixed` 없이 `precision(4)` → `33.57`
- 이게 `fixed` + `precision(2)`의 결과(`33.57`)와 **똑같다.**

그래서 "어? 비슷한데?" 싶었던 것. 하지만 숫자를 바꿔보면 바로 드러난다.

```cpp
double b = 1234.5678;

cout << fixed; cout.precision(2);   // 1234.57
// fixed 빼면
cout.precision(2);                  // 1.2e+03   ← 지수 표기로 튐
```

유효숫자가 부족하면 **지수 표기(`e+03`)로 넘어가버린다.** 알고리즘 문제에서 이러면 바로 오답.

---

### ④ 실전: `fixed`가 필수인 이유

소수점 아래 자릿수를 맞춰야 하는 문제(**대부분의 실수 출력 문제**)에서는 **`fixed`가 필수**다. 없으면 값에 따라 표기가 제멋대로 바뀌어서(지수 표기 등) 채점이 틀린다.

> 💡 **실전 공식:** 실수 출력은 항상 `cout << fixed;` 먼저 깔고 `cout.precision(n);` 으로 소수점 자릿수 지정.

---

### ⑤ `(double)total / 2` — 굳이 형변환이 필요했을까?

```cpp
double average = (double)total / 2;
```

> **의문:** "굳이 `(double)`을 넣을 필요가 있었나? 어차피 결과를 `double average`에 담고, 하나가 실수면 자연스럽게 double로 바뀔 텐데?"

**결론: 넣어야 한다. 안 넣으면 오답이 난다.** `total`도 정수, `2`도 정수라서 `total / 2`는 **정수 나눗셈**으로 먼저 계산돼 소수점이 버려진다. 그 버려진 값을 나중에 `double`에 담아봐야 이미 잃어버린 `.5`는 돌아오지 않는다.

```cpp
int total = 35;
double average = total / 2;          // 35/2 = 17 (정수나눗셈) → 17.0  ❌
double average = (double)total / 2;  // 35.0/2 = 17.5           → 17.5 ✅
```

> ⚠️ **핵심:** 나눗셈은 **대입(=)보다 먼저** 일어난다. double로 바꾸려면 나눗셈 **전에** 한쪽을 실수로 만들어야 한다.

---

### ⑥ `/`는 양쪽 타입에 따라 두 얼굴

`/`가 원래부터 "몫만" 구하는 게 아니라, **양쪽이 무슨 타입이냐에 따라** 다르게 행동한다.

```cpp
35   / 2    →  17      // 정수 ÷ 정수 → "정수 나눗셈" → 몫만
35.0 / 2    →  17.5    // 실수가 끼면 → "실수 나눗셈" → 소수점까지
```

같은 `/` 기호인데 결과가 다르다. C++이 양쪽 타입을 보고 *"정수끼리네? 그럼 몫만"* / *"실수가 있네? 그럼 제대로 나눠"* 하고 판단한다.

---

### ⑦ 왜 이렇게 나뉘나 — int와 double은 담는 그릇이 다르다

- **`int`** 은 소수점을 담을 자리가 **아예 없는 그릇**이다. `17.5`를 넣으려 해도 `.5`를 담을 칸이 없어서 버린다. 그래서 정수끼리 나누면 어차피 `.5`를 못 담으니 몫만 남긴다.
- **`double`** 은 소수점 자리가 있는 그릇이다. `35.0`의 `.0`이 바로 *"나 소수점 담을 수 있어"* 라는 표시다. 그래서 `35.0`이 한쪽에 있으면 C++은 *"결과도 소수점 담을 수 있겠네"* 하고 실수 나눗셈을 한다.

> 💡 `.0`이라는 자리가 있으니까, 나머지를 버리지 않고 소수점으로 표현하는 것이다.
