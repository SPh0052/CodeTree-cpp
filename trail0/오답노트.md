## Lesson 1 — 변수 · char/string · cout (2026-07-14)

### ① 문자(char) vs 문자열(string)

```cpp
char a = 'T';      // 문자 하나 → 작은따옴표 ' '
string a = "T";    // 문자열   → 큰따옴표 " "
```

- **문자(char)** 는 딱 한 글자, **작은따옴표** `'T'`
- **문자열(string)** 은 0글자 이상, **큰따옴표** `"T"`

---

### ② 같은 이름으로 변수를 두 번 선언하면 안 되는 이유

#### ❌ 틀린 코드

```cpp
int main() {
    char a = 'C';
    char a = 'T';   // ❌ 컴파일 에러: redeclaration of 'char a'
    cout << a;
    return 0;
}
```

#### 🐛 왜 안 될까?

`char a = 'C';` 이 한 줄은 사실 두 가지 일을 한다.

1. **선언(declaration)** — "메모리에 `a`라는 이름의 char 공간을 하나 잡아라"
2. **초기화(initialization)** — "거기에 `'C'`를 넣어라"

그런데 다음 줄에서 또 `char a`라고 하면, 컴파일러 입장에선 **이미 있는 이름으로 또 공간을 잡으라**는 소리가 된다. 그럼 나중에 `a`라고 썼을 때 둘 중 뭘 가리켜야 할지 알 수 없다. 그래서 아예 금지.

#### ✅ 값을 바꾸고 싶으면 (선언은 한 번만, 대입은 여러 번)

```cpp
int main() {
    char a = 'C';   // 선언 + 초기화 (한 번만)
    a = 'T';        // 재대입 (type 없이!)
    cout << a;      // T
    return 0;
}
```

> 💡 **핵심:** `char`(타입)는 **선언할 때 딱 한 번**만 쓴다. 값만 바꿀 땐 타입 없이 `a = 'T';`

### ③ 변수 값 대입 순서 이해하기

```cpp
#include <iostream>
using namespace std;

int main() {
    int a = 1;
    int b = 5;
    int c = 3;

    a = c;      // a = 3        (a에 c 값 복사)
    a = a + c;  // a = 3 + 3 = 6
    b = b - c;  // b = 5 - 3 = 2

    cout << a << "\n";  // 6
    cout << b << "\n";  // 2
    cout << c << "\n";  // 3  (c는 안 바뀜)
    return 0;
}
```

> 💡 대입(`=`)은 **오른쪽을 먼저 계산해서 왼쪽에 넣는다.** 윗줄부터 순서대로 값이 갱신된다.

### ④ C++ `cout` vs C `printf` — 설계 철학의 차이

> **의문:** "C++은 변수랑 줄바꿈, 문자를 한 따옴표 안에 동시에 표현할 수 없구나. C언어(`printf`)는 됐는데?"

정확히 이해한 것. `cout`은 그런 방식이 **아니다.**

| | C `printf` | C++ `cout` |
|---|---|---|
| 방식 | **틀(서식 문자열)** 을 주고 **구멍(`%d`)** 에 값을 꽂음 | 출력할 것들을 `<<`로 **순서대로 나열** |
| 예시 | `printf("%d\n", x);` | `cout << x << "\n";` |
| 타입 검사 | ❌ 못 잡음 (`%d`에 실수 넣어도 통과) | ✅ 컴파일러가 타입 보고 알아서 처리 |
| 서식 문자 | `%d %f %lf %c %s`… 외워야 함 | 필요 없음 |

**`cout`이 이렇게 설계된 이유:**
`printf`는 `%d`인데 실수를 넣거나 `%s`인데 정수를 넣어도 **컴파일러가 못 잡아낸다.** 그래서 실행 중에 이상한 값이 나오거나 프로그램이 터진다. 반면 `cout << x`는 컴파일러가 `x`의 타입을 보고 알아서 처리하니 그런 사고가 안 나고, 서식 문자(`%d`, `%f`, `%lf`, `%c`, `%s`…)를 외울 필요도 없다.
